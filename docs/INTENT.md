# Intent

## 何を達成したいか

Github PRレビューに変わる最高にイケてる開発ツールを作りたい。

解決したいペイン
- 1行1行コードをレビューすること
- 変更の中身を理解せず、現場ネコスタイルでNo Look Approveすること
- 人間がボトルネックとなり、生産性向上の恩恵を受けられないこと
- PRのタイトル・本文が実際の変更内容とは一致しない可能性があること

## どうやって達成するか

### レビューの支援

以下のようなレビューを支援する機能を提供したい

- 変更の影響範囲・リスクの把握
- 変更内容について詳しくみたい時だけ、その変更内容に関連するファイル差分だけが見れる
- レビューパターンを記憶し、これまでのレビューから想定されるコメントをsuggestする

### ローカルベースのGUIツール

以下3つを同じ場所で画面の切り替えや移動なく行えるようにしたい

- ローカルのworktreeで作業するAgentの作業に対するレビュー
- Devin/CopilotなどのPRのレビュー
- 同僚のPRのレビュー

### オートメーション

リスクや影響範囲に応じて、自動的にapproveすることができるようにしたい。
また、コーディングエージェントによるPRであればenable auto mergeも行い、マージすることができるようにしたい。

リスクや影響範囲の定義はユーザーにカスタマイズさせることを許しつつ、デフォルトの定義を用意する。
オートメーションもデフォルトは無効で有効にした場合は、リスクレベルをユーザーが選んで自動化する。

### 開発の支援

また、コードベースからTODOやRoadmapを読み取り、開発タスクを提案することができるようにしたい。
提案した開発タスクはワンクリックで、worktreeを作成して開発を始めることができるようにしたい。
